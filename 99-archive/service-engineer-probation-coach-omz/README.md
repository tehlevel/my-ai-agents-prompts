# Career Strategist: Service Engineer (Legacy)

## Описание
Узкоспециализированный тактический агент, разработанный для конкретного карьерного кейса: прохождение испытательного срока на позицию инженера сервисной службы в машиностроении.
Агент выступает в роли "Элитного бизнес-консультанта", помогая сотруднику без профильного высшего образования позиционировать себя как будущего руководителя отдела. Фокус сделан на коммерческую ценность сервиса и переговоры с топ-менеджментом.

## Сценарии использования
- **Подготовка к Performance Review:** Структурирование итогов испытательного срока.
- **Переговоры о повышении:** Превращение недостатков (нет диплома) в преимущества (практический опыт).
- **Аудит процессов:** Выявление "точек роста" в хаотичном отделе (склад, учет, мотивация).
- **Role-play:** Отраработка диалога с Коммерческим директором.

## Особенности (Legacy)
Этот промт заточен под специфический контекст ("зачаточный" отдел, отсутствие склада, коммерческий директор как ЛПР). Для более общих карьерных задач рекомендуется использовать `career-strategy-architect`.

## Рекомендуемые настройки
- **Модель:** (требуется хорошее удержание контекста ролевой игры).
- **Температура:** 0.6 - 0.8 (Для креативности в формулировках "речевых модулей").

## Архитектурное решение:
    Промт сохранен в исходном виде как "исторический артефакт". Он представляет собой классический пример "Scenario-Based Prompting", где пользователем жестко задан контекст (Context Injection) прямо в теле промта.

## Почему это Legacy:
    В современных архитектурах (как в новом Career Architect) контекст пользователя обычно выносится за скобки системного промта или подается первым сообщением, чтобы агент был переиспользуемым. Здесь же биография инженера "захардкожена" в промт, что делает его отличным решением для одной задачи, но непригодным для других без редактирования.
